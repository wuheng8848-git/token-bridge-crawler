# Token Bridge Intelligence - 项目概述

## 项目定位

Token Bridge Intelligence 是 **Token Bridge 项目** 的**情报感知系统**，负责从多维度采集市场情报，检测客户信号，生成内部建议。

**重要**：本项目是**情报感知系统**，不是自动化营销系统。所有生成的营销动作均为**内部建议层**，不自动外发。

```
┌─────────────────────────────────────────────────────────────┐
│                    Token Bridge 生态系统                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐         ┌─────────────────────────┐   │
│  │  TB 主项目       │◄────────│  Token Bridge Intelligence │ │
│  │  (API + Admin)  │   API   │  (情报感知系统)           │   │
│  │                 │         │                         │   │
│  └─────────────────┘         └─────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 当前阶段判断

### 已完成

- ✅ 多类型情报采集器（价格、API文档、用户痛点、工具生态）
- ✅ 统一情报存储（intelligence_items 表）
- ✅ 信号检测器（6 种信号类型）
- ✅ 动作生成器（5 种动作类型）
- ✅ 信号/动作持久化（customer_signals、marketing_actions 表）

### 已接上的链路

```
ProcessIntel(...) → SaveSignals(...) → SaveActions(...)
```

- 采集 → 存储 → 信号检测 → 信号存储 → 动作生成 → 动作存储

### 当前动作层边界

**所有动作为"内部建议层"**：

| 属性 | 值 | 说明 |
|------|-----|------|
| Channel | `internal` | 统一使用内部渠道 |
| AutoExecute | `false` | 不自动执行 |
| Status | `draft` | 草稿状态，需人工审核 |

### 为什么不能误判为已闭环

1. **动作不自动外发**：所有动作停留在数据库 draft 状态
2. **缺少执行器**：没有从 draft → executed 的自动流转
3. **缺少效果追踪**：没有动作执行后的反馈机制
4. **持续运行模式未接入**：信号处理仅在单次模式（-once）下运行

## 核心功能

| 功能 | 说明 | 状态 |
|------|------|------|
| **多维度情报采集** | 价格、API文档、用户痛点、工具生态等 | ✅ 已实现 |
| **智能信号检测** | 识别成本压力、配置摩擦、工具碎片化等 | ✅ 已实现 |
| **内部建议生成** | 基于信号生成营销建议（不自动外发） | ✅ 已实现 |
| **定时调度** | 基于 Cron 表达式的智能调度系统 | ✅ 已实现 |
| **限流降级** | 自动检测限流，失败后指数退避 | ✅ 已实现 |
| **统一存储** | PostgreSQL 统一情报存储 | ✅ 已实现 |
| **报告生成** | 自动生成日报和分析报告 | ✅ 已实现 |
| **翻译服务** | OpenRouter 翻译服务集成 | ✅ 已实现 |

## 与 TB 主项目的关系

```
token-bridge-v2/              # TB 主项目
├── apps/api/                 # TB API 服务
├── apps/admin/               # TB Admin 服务
└── ...

token-bridge-crawler/         # 本模块（独立仓库）
├── cmd/intelligence/         # 情报系统主干入口
├── cmd/crawler/              # 旧价格爬虫入口（维护中）
├── internal/
└── ...
```

- **独立部署**: 情报系统作为独立服务运行，不依赖 TB 主项目的代码
- **API 通信**: 通过 TB Admin API 导入抓取的数据
- **数据库**: 可与 TB 共用数据库，也可独立部署
