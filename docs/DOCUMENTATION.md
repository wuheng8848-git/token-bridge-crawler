# 文档规范

> 本规范适用于 Token Bridge Intelligence 项目，采用**渐进式披露**原则。

---

## 一、核心原则

### 1.1 渐进式披露

```
Level 0: 一句话定位     → README.md 第一行
Level 1: 一页纸概览     → index.md
Level 2: 快速参考       → docs/00-quick-reference.md
Level 3: 详细文档       → docs/01-07.md
Level 4: 深度资料       → 源码、迁移文件、决策记录
```

**读者路径**：
```
新人 → Level 0 → Level 1 → Level 2 → 按需深入
AI   → Level 1 → Level 2 → Level 3
```

### 1.2 文档分层

| 层级 | 文档 | 受众 | 更新频率 | 长度限制 |
|------|------|------|----------|----------|
| **入口层** | README.md, index.md | 所有人 | 低 | < 200 行 |
| **速查层** | 00-quick-reference.md | 开发者 | 中 | < 100 行 |
| **概念层** | 01-overview, 02-architecture | 新成员 | 中 | < 300 行 |
| **操作层** | 03-development, 04-deployment | 开发者 | 高 | < 400 行 |
| **参考层** | 05-api, 07-signal-model | 日常查阅 | 中 | 不限 |
| **记录层** | CHANGELOG.md, decisions/ | 架构师 | 低 | 按需增长 |

| 目录 | 内容 | 受众 | 更新频率 |
|------|------|------|----------|
| **product/** | 产品功能、用户指南 | 产品、运营、新成员 | 低 |
| ↳ **00-product-charter.md** | ⭐ 产品总纲（最高优先级） | 产品、架构师 | 极低 |
| ↳ **PRD-*.md** | 产品需求文档 | 产品、开发 | 按需 |

| **deployment/** | 部署、配置、运维 | 运维、开发者 | 中 |

## 五、更新规范

### 5.1 必须更新

| 场景 | 更新文档 |
|------|----------|
| **新功能开发前** | **必须先创建 PRD** → docs/product/PRD-{功能名}.md |
| 新增功能（开发中） | CHANGELOG.md + docs/product/02-features.md |
| 架构变更 | docs/architecture/ + docs/progress/decisions/ |
| API 变更 | docs/architecture/05-api.md |
| 数据结构变更 | docs/architecture/02-data-model.md |
| 新增采集器 | docs/architecture/03-collectors.md |
| 新增翻译器 | docs/architecture/04-translator.md |
| 部署变更 | docs/deployment/ |
| 版本发布 | CHANGELOG.md + docs/progress/02-changelog.md |

### 5.2 PRD 开发流程

```
0. 确认产品总纲 → 1. 创建 PRD → 2. 评审确认 → 3. 开发实现 → 4. 验收检查 → 5. 归档 PRD
```

**强制规则**：
- ❌ 禁止无产品总纲就开始规划功能
- ❌ 禁止 PRD 与产品总纲冲突
- ❌ 禁止无 PRD 直接开发新功能
- ❌ 禁止 PRD 未评审就开发
- ✅ PRD 状态变更需同步更新
- ✅ 产品边界变更需更新总纲

### 5.3 更新检查清单

```markdown
- [ ] 文档标题是否准确
- [ ] 最后更新日期是否修改
- [ ] 相关文档链接是否更新
- [ ] index.md 索引是否需要更新
- [ ] 代码示例是否可运行
- [ ] 数据统计是否最新
```

### 5.4 文档版本号

每个文档底部标注版本：

```markdown
**最后更新**：YYYY-MM-DD
**版本**：vX.Y
```

- **主版本（X）**：重大重构
- **次版本（Y）**：内容更新
