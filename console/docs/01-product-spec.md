# 爬虫控制台产品文档

## 1. 产品定位

爬虫控制台是 Token Bridge Intelligence 情报系统的**技术运营界面**，面向开发和运维人员，提供情报采集的监控、调试和配置能力。

**与主项目 Admin 的分工**:
| 系统 | 面向用户 | 核心职责 |
|------|----------|----------|
| 主项目 Admin | 运营/产品 | 情报处理、业务决策、日报分析 |
| 爬虫控制台 | 开发/运维 | 采集监控、技术调试、配置管理 |

---

## 2. 功能模块

### 2.1 监控大盘 (Dashboard)

**目标**: 一眼看清情报系统整体健康状况

**功能**:
- **采集器状态卡片**: 各采集器运行状态、成功率、最近运行时间
- **情报数量趋势**: 按类型/来源的 24h/7d/30d 趋势图
- **信号检测统计**: 各类型信号数量、检测成功率
- **系统健康度**: 翻译服务状态、数据库存储状态

**数据示例**:
```
┌─────────────────────────────────────────────────────────────┐
│  采集器状态                    │  今日情报统计                │
│  ┌──────────┬────────┬──────┐ │  ┌──────────┬──────────────┐│
│  │ Google   │ ✅ 正常 │ 2min │ │  │ 价格情报   │ 156 条       ││
│  │ OpenAI   │ ✅ 正常 │ 5min │ │  │ 用户痛点   │ 23 条        ││
│  │ HN       │ ⚠️ 降级 │ 1h   │ │  │ 社区动态   │ 89 条        ││
│  └──────────┴────────┴──────┘ │  └──────────┴──────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 2.2 采集器管理 (Collectors)

**目标**: 管理所有情报采集器的运行和配置

**功能**:
- **采集器列表**: 名称、类型、状态、最后运行时间、成功率
- **启停控制**: 启用/禁用单个采集器
- **频率配置**: 调整采集间隔（Cron 表达式）
- **运行日志**: 查看最近采集日志、错误详情
- **手动触发**: 立即执行一次采集（用于测试）

**采集器类型**:
| 类型 | 采集器 | 状态 |
|------|--------|------|
| 价格 | Google/OpenAI/Anthropic/OpenRouter | 可用 |
| API文档 | OpenAI | 可用 |
| 用户痛点 | HackerNews/Reddit/ConfigPain | 可用 |
| 工具生态 | Ecosystem | 可用 |
| 社区 | Discord/LinkedIn/ProductHunt | 部分模拟 |

---

### 2.3 情报浏览 (Intelligence)

**目标**: 查看原始采集的情报数据

**功能**:
- **情报列表**: 分页展示 `intelligence_items`
- **筛选条件**: 按类型、来源、时间范围、状态筛选
- **详情查看**: 原始内容、翻译内容、元数据
- **关联信号**: 查看该情报检测到的信号
- **重新翻译**: 手动触发翻译（用于翻译失败时）

**列表字段**:
- ID、情报类型、来源、标题（中文）、采集时间、状态

---

### 2.4 信号调试 (Signals)

**目标**: 调试信号检测器的准确性和阈值

**功能**:
- **信号列表**: 查看 `customer_signals` 所有信号
- **筛选条件**: 按信号类型、强度、平台、状态筛选
- **详情查看**: 关联情报、检测依据、置信度
- **人工标记**: 标记信号有效性（用于训练 Detector）
- **阈值调整**: 调整信号检测的敏感度

**信号类型**:
| 信号 | 优先级 | 说明 |
|------|--------|------|
| cost_pressure | P0 | 成本压力 |
| config_friction | P0 | 配置摩擦 |
| tool_fragmentation | P0 | 工具碎片化 |
| governance_start | P1 | 治理起点 |
| migration_intent | P1 | 迁移意愿 |
| general_interest | P3 | 泛兴趣 |

---

### 2.5 翻译服务 (Translation)

**目标**: 监控和管理翻译服务

**功能**:
- **服务状态**: 各翻译供应商状态（火山/百度）
- **用量统计**: 本月调用量、成功率、平均耗时
- **队列监控**: 待翻译任务数、处理速度
- **故障切换**: 手动切换翻译供应商
- **失败重试**: 批量重试翻译失败的数据

**翻译供应商优先级**:
```
火山引擎（200万/月，172ms）→ 百度大模型（100万/月）→ 百度通用（5万/月）
```

---

### 2.6 系统配置 (Settings)

**目标**: 管理爬虫系统的运行配置

**功能**:
- **调度配置**: 全局 Cron 表达式、并发数限制
- **限流配置**: 各厂商请求间隔、重试策略
- **告警配置**: 采集失败告警阈值、通知方式
- **存储清理**: 历史数据保留策略
- **环境变量**: 查看当前生效的配置（脱敏）

## 4. 权限设计

| 角色 | 权限 |
|------|------|
| **管理员** | 全部功能，包括配置修改 |
| **开发** | 查看、调试、手动触发，不可修改配置 |
| **运维** | 查看、监控、启停采集器，不可修改代码逻辑 |

---

## 5. 技术约束

- **复用主项目 Admin 技术栈**: React + react-admin + MUI
- **复用主题系统**: 保持视觉一致性
- **独立部署**: 可独立运行，也可嵌入主项目
- **API 直连**: 直接调用爬虫系统的 Go API
